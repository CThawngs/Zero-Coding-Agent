### 1. PARADIGM SHIFT: SYNTAX → INTENT

**[ENGLISH VERSION]**

**Traditional coding:** Developer writes exact syntax (braces, semicolons, types).  
**New paradigm:** Developer expresses *what* to build; machine handles *how*.  
**Intent-driven development** = "Vibe Coding".  
Friction of translation (human → abstract → syntax) is collapsing.  
**Most significant transformation since high-level languages.**

**[VIETNAMESE VERSION]**

**Lập trình truyền thống:** Dev viết chính xác cú pháp (ngoặc, dấu chấm phẩy, kiểu dữ liệu).  
**Paradigm mới:** Dev diễn đạt *cái gì* cần xây; máy lo *như thế nào*.  
**Phát triển theo ý định** = "Vibe Coding" (Lập trình theo cảm hứng/luồng).  
Ma sát quá trình dịch (con người → trừu tượng → cú pháp) đang sụp đổ.  
**Biến đổi lớn nhất kể từ ngôn ngữ bậc cao.**

### 2. VIBE CODING vs AGENTIC ENGINEERING (THE SPECTRUM)

**[ENGLISH VERSION]**

```
VIBE CODING ←────────────────────────────────→ AGENTIC ENGINEERING
```

| Dimension | Vibe Coding | Agentic Engineering |
|---|---|---|
| **Goal** | Rapid prototyping | Production systems |
| **Upfront Investment (CapEx)** | Low | High |
| **Ongoing Cost (OpEx)** | High (technical debt, bugs) | Low (automated quality gates) |
| **Discipline** | Intuition, flow | Evaluation, constraints, harnesses |
| **Best For** | Exploration, demos, learning | Critical systems, scale, teams |

**Key Insight:** Most teams start with vibe coding, mature to agentic engineering. The spectrum is not binary.

### 4. FACTORY MODEL & DEVELOPER AS ORCHESTRATOR

**[ENGLISH VERSION]**

**Core Metaphor:** Don't build the car. Build the **factory that builds cars**.  
**Developer Role:** System orchestrators who design evaluation, constraint, and context harnesses.  
**Harness (Framework):** The evaluation, constraint, and context systems that safely guide autonomous execution.

**[VIETNAMESE VERSION]**

### 5. CONTEXT ENGINEERING = REAL SKILL

**[ENGLISH VERSION]**

> **"Context engineering is the new prompt engineering"**

**What is Context Engineering?**
- Dynamic assembly of relevant information for each agent turn
- Includes: Codebase, docs, user intent, tools, memory, policies, examples
- **Not** static prompts — living, breathing context

**Three Layers:**
1. **Static Context:** System prompt, coding standards, architecture docs
2. **Dynamic Context:** Current task, relevant files, recent changes
3. **Episodic Context:** Past interactions, decisions, feedback

**Skills to Master:**
- Context window management (what to include/exclude)
- Retrieval strategies (RAG for codebases)
- Context compression (summarization vs full inclusion)

**[VIETNAMESE VERSION]**

> **"Kỹ thuật ngữ cảnh là kỹ thuật prompt mới"**

**Kỹ thuật ngữ cảnh là gì?**
- Tự động lắp ráp thông tin liên quan cho mỗi lượt tác nhân
- Bao gồm: Codebase, tài liệu, ý định người dùng, công cụ, bộ nhớ, chính sách, ví dụ
- **Không** phải prompt tĩnh — ngữ cảnh sống, thở

**3 Lớp Ngữ Cảnh:**
1. **Ngữ cảnh tĩnh:** System prompt, tiêu chuẩn code, tài liệu kiến trúc
2. **Ngữ cảnh động:** Nhiệm vụ hiện tại, file liên quan, thay đổi gần đây
3. **Ngữ cảnh sự kiện:** Tương tác quá khứ, quyết định, phản hồi

**Kỹ năng cần thành thạo:**
- Quản lý cửa sổ ngữ cảnh (bao gồm gì/loại gì)
- Chiến lược truy xuất (RAG cho codebase)
- Nén ngữ cảnh (tóm tắt vs bao gồm toàn bộ)

### 6. THE NEW SDLC: AI TRANSFORMS EACH PHASE

**[ENGLISH VERSION]**

| Traditional SDLC Phase | AI Transformation | Key Change |
|---|---|---|
| **Requirements & Planning** | Intent specification, auto-generated specs | Natural language → structured requirements |
| **Design & Architecture** | AI-assisted design, constraint validation | Architecture as executable constraints |
| **Implementation** | Autonomous code generation, multi-file edits | Developer reviews, doesn't write |
| **Testing & QA** | AI-generated tests, property-based testing | Continuous validation, LLM-as-Judge |
| **Code Review** | Automated review, policy enforcement | Human reviews intent, not syntax |
| **Deployment** | Progressive rollout, automated rollback | Harness-controlled releases |
| **Maintenance** | Self-healing, dependency updates | Agent monitors & fixes |

**Compression Effect:** What took weeks → hours. But requires new quality gates.

| Giai đoạn SDLC Truyền Thống | Biến Đổi Bởi AI | Thay Đổi Chính |
|---|---|---|

| **Thiết kế & Kiến trúc** | Thiết kế có AI hỗ trợ, xác thực ràng buộc | Kiến trúc như ràng buộc có thi hành được |
| **Triển khai** | Tự động tạo code, chỉnh sửa đa file | Dev review, không viết |

**Hiệu ứng nén:** Cái trước mất tuần → giờ. Nhưng cần cổng chất lượng mới.

### 7. FACTORY MODEL: HARNESS ENGINEERING

**[ENGLISH VERSION]**

**Four Harness Types (Mapped to SDLC):**

| # | Harness | SDLC Phase | Purpose |
|---|---|---|---|
| **1** | **Requirements/Planning/Architecture** (Configuring) | Pre-execution | Define intent, constraints, success criteria |
| **2** | **Implementation** (Running) | During execution | Sandbox, tool access, step validation |
| **3** | **Testing & QA** (Feedback Loop) | Post-execution | Automated eval, LLM-as-Judge, regression detection |
| **4** | **Review/Deploy/Maintain** (Observing) | Continuous | Observability, rollback, policy enforcement |

**What's IN the Harness?**
- Evaluation criteria: correctness, style, security, performance
- Constraints: file access, API calls, cost limits, timeouts
- Context providers: codebase index, docs, standards
- Observability: logging, tracing, metrics
- Human-in-the-loop checkpoints

**[VIETNAMESE VERSION]**

**4 Loại Khung (Ánh xạ SDLC):**

| # | Khung (Harness) | Giai đoạn SDLC | Mục đích |
|---|---|---|---|
| **1** | **Yêu cầu/Lập kế hoạch/Kiến trúc** (Cấu hình) | Trước thực thi | Định nghĩa ý định, ràng buộc, tiêu chí thành công |
| **2** | **Triển khai** (Chạy) | Khi thực thi | Sandbox, truy cập công cụ, xác thực bước |
| **3** | **Test & QA** (Vòng lặp phản hồi) | Sau thực thi | Đánh giá auto, LLM裁判, phát hiện hồi quy |
| **4** | **Review/Triển khai/Bảo trì** (Quan sát) | Liên tục | Quan sát, rollback, thi hành chính sách |

**Khung làm việc chứa gì?**
- Tiêu chí đánh giá: đúng đắn, phong cách, bảo mật, hiệu năng
- Ràng buộc: truy cập file, gọi API, giới hạn chi phí, timeout
- Cung cấp ngữ cảnh: index codebase, tài liệu, tiêu chuẩn
- Quan sát: log, trace, metric
- Điểm kiểm tra có người trong vòng lặp

### 8. DEVELOPER'S EVOLVING ROLE

**[ENGLISH VERSION]**

| Mode | Name | Characteristics |
|---|---|---|
| **Conductor** | Hands-on, real-time | Directing single agent, reviewing each step, immediate feedback |
| **Orchestrator** | Async, multi-agent | Delegating to agent swarm, reviewing batch results, strategic oversight |

**The 80% Problem:** Agents handle 80% of routine work well. The remaining 20% (edge cases, architecture, judgment) needs human. **Don't automate the 20% poorly.**

**[VIETNAMESE VERSION]**

| Chế độ | Tên | Đặc điểm |
|---|---|---|
| **Conductor** (Người 지휘) | Trực tiếp, thời gian thực | Điều khiển 1 tác nhân, review mỗi bước, phản hồi ngay |
| **Orchestrator** (Người điều phối) | Bất đồng bộ, đa tác nhân | Ủy thác cho đàn tác nhân, review kết quả lô, giám sát chiến lược |

**Vấn đề 80%:** Tác nhân xử lý tốt 80% việc thường quy. 20% còn lại (trường hợp biên, kiến trúc, phán đoán) cần con người. **Đừng tự động hóa kém 20% đó.**

### 9. CODING AGENTS IN PRACTICE

**[ENGLISH VERSION]**

**Where Agents Fit in Developer's Day:**
- Boilerplate generation → Agent
- Refactoring → Agent
- Test writing → Agent
- Documentation → Agent
- **Architecture decisions → Human**
- **User research → Human**
- **Code review (intent) → Human**

**Vibe Coding vs Production-Ready Agents:**
- Vibe: "Make this work" → copy-paste → ship
- Production: "Make this work within constraints" → eval harness → CI gate → deploy

**[VIETNAMESE VERSION]**

**Tác nhân phù hợp đâu trong ngày Dev:**
- Sinh boilerplate → Tác nhân
- Tái cấu trúc → Tác nhân
- Viết test → Tác nhân
- Tài liệu hóa → Tác nhân
- **Quyết định kiến trúc → Con người**
- **Nghiên cứu người dùng → Con người**
- **Review code (ý định) → Con người**

**Vibe Coding vs Tác nhân Production-Ready:**
- Vibe: "Làm cho chạy" → copy-paste → triển khai
- Production: "Làm chạy trong ràng buộc" → khung đánh giá → cổng CI → triển khai

### 10. ECONOMICS OF AI DEVELOPMENT

**[ENGLISH VERSION]**

| Metric | Vibe Coding | Agentic Engineering |
|---|---|---|
| **CapEx (Upfront)** | Low (just start coding) | High (build harnesses) |
| **OpEx (Ongoing)** | High (debugging, debt, review) | Low (automated quality gates) |
| **Scaling** | Linear with team size | Exponential (more agents = more leverage) |
| **Risk** | Technical debt accumulation | Controlled via harnesses |

**Context Engineering as Financial Lever:**
- Better context → fewer tokens → lower cost
- Dynamic context → right info at right time → fewer errors
- Context compression → fit more in window → better results

**Intelligent Model Routing:** Route simple tasks to cheap models, complex to expensive.

**[VIETNAMESE VERSION]**

| Chỉ số | Vibe Coding | Agentic Engineering |
|---|---|---|
| **CapEx (Ban đầu)** | Thấp (chỉ việc bắt đầu code) | Cao (xây khung) |
| **OpEx (Vận hành)** | Cao (debug, nợ, review) | Thấp (cổng chất lượng auto) |
| **Quy mô** | Tuyến tính theo team | Mũ (nhiều tác nhân = đòn bẩy hơn) |
| **Rủi ro** | Tích lũy nợ kĩ thuật | Kiểm soát qua khung |

**Kỹ thuật ngữ cảnh như đòn bẩy tài chính:**
- Ngữ cảnh tốt hơn → ít token hơn → chi phí thấp hơn
- Ngữ cảnh động → đúng info đúng lúc → ít lỗi hơn
- Nén ngữ cảnh → nhét nhiều vào cửa sổ → kết quả tốt hơn

**Định tuyến mô hình thông minh:** Chuyển task đơn giản cho mô hình rẻ, phức tạp cho mô hình đắt.

## 📝 SUMMARY (TÓM TẮT)

### [ENGLISH VERSION]

This whitepaper defines the **paradigm shift** from syntax-driven to **intent-driven development** ("Vibe Coding"). It introduces a **spectrum** from rapid vibe coding (low CapEx, high OpEx/debt) to disciplined **Agentic Engineering** (high CapEx/harness investment, low OpEx). The core framework is the **Factory Model**: developers become **orchestrators** who build **harnesses** (evaluation, constraint, context, observability) that safely guide autonomous agents. **Context Engineering** replaces prompt engineering as the critical skill. AI compresses the traditional **SDLC** across all 7 phases. The economics favor Agentic Engineering at scale. Start by building personal eval harnesses and practicing context engineering daily.

### [VIETNAMESE VERSION]

Whitepaper này định nghĩa **chuyển đổi mô hình** từ phát triển theo cú pháp sang **phát triển theo ý định** ("Vibe Coding"). Nó giới thiệu một **spectrum** từ vibe coding nhanh (CapEx thấp, OpEx/nợ cao) đến **Agentic Engineering** có kỷ luật (CapEx/đầu tư khung cao, OpEx thấp). Khung cốt lõi là **Mô hình Nhà máy**: dev trở thành **người điều phối** xây dựng **khung làm việc** (đánh giá, ràng buộc, ngữ cảnh, quan sát) hướng dẫn an toàn các tác nhân tự động. **Kỹ thuật Ngữ cảnh** thay thế kỹ thuật prompt thành kỹ năng quan trọng. AI nén **SDLC** truyền thống qua 7 giai đoạn. Kinh tế ủng hộ Agentic Engineering ở quy mô lớn. Bắt đầu bằng việc xây khung đánh giá cá nhân và luyện ngữ cảnh hàng ngày.
